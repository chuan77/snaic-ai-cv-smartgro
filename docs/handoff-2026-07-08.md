# Project Handoff — 2026-07-08

## What This Project Does

SmartCart AI Checkout Assistant: a 5-stage pipeline (Days 1-5) that turns unannotated grocery classification photos into a working YOLO11 object detector by programmatically compositing product crops onto synthetic backgrounds, then serves live detections through a checkout UI that prices items from a catalog. Two serving surfaces exist: a Gradio dashboard (`main_day5_deploy.py`, Python-only) and a React/TypeScript/Vite frontend (`frontend/`) backed by a FastAPI service (`main_api_server.py`). All ML code targets Apple Silicon (`torch.device("mps" if torch.backends.mps.is_available() else "cpu")`). See `CLAUDE.md` and `docs/handoff-2026-07-07.md` for prior context (dataset/detector expansion, the initial FastAPI+React wiring).

## What Was Completed Today

### 1. Fixed the frontend's single-scan-per-session bug

- **Root cause**: `frontend/src/components/CameraFeed.tsx` conditionally rendered either a live `<Webcam ref={webcamRef}>` or a static `<img>` depending on whether `feedImage` was set. The first "Freeze & Detect" click set `feedImage`, unmounting `<Webcam>` and detaching `webcamRef` permanently — every later click's `webcamRef.current?.getScreenshot()` silently returned `undefined` and no-opped. The only way back to a live camera was "Reset," which wiped the whole cart, not just the frozen frame.
- Cart state itself (`useSmartCart.ts`'s `cartLines` array) was already correct and needed no redesign — it already appended rather than overwrote.
- Fix: added a `resumeLive()` store action (clears `feedImage`/`detections`/`detectionAccents`, leaves `cartLines` untouched) and a ~1s auto-resume timer in `CameraFeed.tsx` after each scan, so the camera goes back live automatically and multiple items can be scanned in sequence with the running total updating correctly.
- Verified end-to-end via a headless Playwright session with mocked `/predict` responses: two consecutive freezes produced two distinct cart lines (Golden Delicious $1.75 + Granny Smith $1.75) summing to $3.50, and Reset correctly emptied the cart with the camera staying live.
- **Note on commit history**: this fix was implemented and verified in-session but not committed by me — the user committed it independently afterward as three commits with generic messages (`717df89`/`c4a167e` "first commit", `ec5f4a1` "forst commit"). No action needed, just explains why those commits look untitled if you're reading `git log`.

### 2. Made the FastAPI backend + React frontend configurable via `.env`

Full subagent-driven-development cycle (design → plan → 4 implementer/reviewer task cycles → final whole-branch review → merge), done in an isolated worktree, merged to `main` as `fa469f0`.

- **Backend** (`src/deploy/api_server.py`, `main_api_server.py`): added `python-dotenv`; six new/newly-wired env vars — `SMARTCART_HOST`, `SMARTCART_PORT`, `SMARTCART_RELOAD`, `SMARTCART_CORS_ORIGINS` (comma-separated), `SMARTCART_WEIGHTS_PATH` (pre-existing, now `.env`-loadable), `SMARTCART_CATALOG_PATH` (new). No settings/config class — plain `os.environ.get(KEY, default)` functions (`get_cors_origins`, `get_catalog_path`, `parse_bool_env`, `get_server_config`), matching the codebase's existing pattern.
- **Non-obvious correctness point**: `load_dotenv()` is called in `src/deploy/api_server.py`, *not* `main_api_server.py`. Because `SMARTCART_RELOAD` defaults to enabling uvicorn's `--reload`, the reload worker imports `src.deploy.api_server:app` directly and never runs `main_api_server.py`'s `__main__` block — loading `.env` only in the entrypoint would leave the reload worker silently running on defaults. Caught by the final whole-branch review; there's now a code comment at the `load_dotenv()` call site explaining this.
- **Frontend** (`frontend/vite.config.ts`): added `VITE_DEV_SERVER_PORT`, read via Vite's built-in `loadEnv()` (no new dependency). `VITE_API_BASE_URL` already existed.

## Running the Stack

```bash
# Install/sync Python deps
uv sync

# Configure (first time only)
cp .env.example .env
cd frontend && cp .env.example .env && cd ..

# Run the backend
uv run python main_api_server.py     # http://localhost:8000 by default

# Run the frontend (separate terminal)
cd frontend && npm install && npm run dev   # http://localhost:5173 by default

# Run tests
uv run pytest -v
cd frontend && npm run build && npm run lint
```
