# Project Handoff — 2026-07-07

## What This Project Does

SmartCart AI Checkout Assistant: a 5-stage pipeline (Days 1-5) that turns unannotated grocery classification photos into a working YOLO11 object detector by programmatically compositing product crops onto synthetic backgrounds, then serves live detections through a checkout UI that prices items from a catalog. Two serving surfaces exist: a Gradio dashboard (`main_day5_deploy.py`, Python-only) and a React/TypeScript/Vite frontend (`frontend/`) backed by a new FastAPI service. All ML code targets Apple Silicon (`torch.device("mps" if torch.backends.mps.is_available() else "cpu")`). See `CLAUDE.md` for the full architecture.

The real Label Studio export turned out to be a single combined `dataset/raw_photos/{images,labels,classes.txt}` covering both categories at brand/SKU granularity (47 classes, e.g. `Ready-To-Eat/Instant-Noodles/Nissin/328-KATONG-LAKSA`), with some photos containing dozens of boxes (flat-lay shots). Reworked `import_label_studio_export` to read `classes.txt`, crop *every* box per label file, and route each crop to a destination folder by keyword match (`"Instant-Noodles"` / `"Chocolate"`) against its class name — collapsing all SKU-level labels into the two planned coarse classes, per an explicit decision to keep the detector at 5 classes rather than expand to ~26 SKU-level ones.
- New category-aware pricing rule `get_base_price` in `src/data/gallery.py` (replacing a hardcoded `"Packages"` substring check): `Packages` → $3.20, `Ready-To-Eat` → $3.00, `Snacks` → $6.20, else $1.75.

- **Gotcha hit and fixed**: a global `~/Library/Application Support/Ultralytics/settings.json` on this machine had `runs_dir` pointing at an unrelated old project (`snaic-ai-programme`). Since `main_day2_train.py` never passed `project=`, the first retrain silently wrote weights outside this repo entirely. Fixed by explicitly passing `project=str(Path("./runs/detect").resolve())`. An orphaned run from before the fix is left at `/Users/chuan/Development/PythonProjects/snaic-ai-programme/runs/detect/{train,val-2}` — outside this repo, not cleaned up.

- Ran the full pipeline: Day 1 catalog rebuild → Day 2 retrain → Day 3 validate → found Chocolate-Bar was the weakest class (mAP50 0.843, recall 0.571) → ran Day 4 stress augmentation targeting `weak_class_id=4` → retrained again → Chocolate-Bar improved to mAP50 0.938, recall 0.8. - **Known, accepted limitation** (not a bug): full-resolution/close-up product photos are out-of-distribution for detection, since Day 2's synthesizer only ever trains on items scaled to ~20-33% of a 640×640 canvas on a plain background. A live webcam capture at normal distance should match the training distribution far better than a macro close-up crop did in ad hoc testing.

## Running the Stack

```bash
# Install/sync Python deps
uv sync

# Run the ML pipeline (only if regenerating data/model)
uv run python main_day1_catalog.py
uv run python main_day2_train.py
uv run python main_day3_fine_tune.py

# Run the backend
uv run python main_api_server.py     # http://localhost:8000

# Run the frontend (separate terminal)
cd frontend && npm install && npm run dev   # http://localhost:5173

# Run tests
uv run pytest -v
```
